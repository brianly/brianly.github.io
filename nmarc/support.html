Support for the NMARC (Unofficial) tool

Builds of the NMARC (Unofficial) tool from the Windows Store are unsupported and used at your own risk. Issues about the tool can be reported in the GitHub issues for the NMARC project and may be addressed in future builds.

It is recommended that you build and run your own instance of the NMARC tool using the code from GitHub for the following reasons:

  • You can audit the source code for yourself.
  • It can run on Windows 7 or places where the Windows Store isn't available.

FAQs

What does the NMARC tool do?

It takes the alignment report in YAML format and converts it to multiple CSV files. This is important because the CSV files can then be filtered with popular tools like Excel.

Does the tool process my data locally?

Yes. The reason for the NMARC tool is to provide a means for you to convert it into CSV locally. This makes it easier to work with in popular tools, and it means that any company or personally identifiable information in the YAML file is kept on your computer. You should obviously use endpoint protection and other measures to protect both the YAML input file and any CSVs that are created.

Why do you recommend that I build and audit my own version?

Good question. It's an open source tool and I'm not assuming responsibility. You should always use software that your IT team has vetted and meets the requirements of your organization. Building your own version is relatively straightforward because it's a simple Windows Forms application.
